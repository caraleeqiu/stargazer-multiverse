# 平行世界追星 (Stargazer Multiverse) PRD

> 版本: v0.1 | 更新时间: 2026-02-22 | 工期: 2天

---

## 一、产品定位

**一句话定义**: 沉浸式互动追星漫画平台 —— 让用户成为故事主角，在平行世界与爱豆相遇。

**核心价值**:
- 陪伴感 (乙女游戏的情感体验)
- 沉浸感 (第一视角 + 选择驱动)
- 创造性 (UGC 内容生态)

---

## 二、用户画像

| 维度 | 描述 |
|------|------|
| 核心用户 | 18-28岁女性，追星/饭圈文化爱好者 |
| 需求痛点 | 想要与爱豆互动的幻想，但现有内容被动消费缺乏代入感 |
| 使用场景 | 睡前、通勤、午休等碎片时间沉浸式娱乐 |

---

## 三、核心功能架构

### 3.1 双线模式

```
┌─────────────────────────────────────────────────────┐
│                   平行世界追星                        │
├──────────────────────┬──────────────────────────────┤
│     内容消费线        │         内容创作线            │
├──────────────────────┼──────────────────────────────┤
│ • 选择世界观          │ • 创建世界观模板              │
│ • 设定人物个性        │ • 编写分支剧情                │
│ • 进入漫画世界        │ • 设计选择节点                │
│ • 关键节点选择        │ • 上传/生成漫画素材           │
│ • 体验平行结局        │ • 发布作品获取收益            │
└──────────────────────┴──────────────────────────────┘
```

### 3.2 用户体验流程

```
选择世界观 → 设定自我角色 → 选择场景/剧本 → 进入漫画
     ↓                                        ↓
 (偶像团、古风、校园、职场...)          沉浸式阅读漫画
                                              ↓
                                     关键节点 → 做选择
                                              ↓
                           不同选择 → 分支剧情 → 平行结局
```

### 3.3 核心交互设计

| 功能模块 | 描述 | 参考 |
|---------|------|------|
| **世界观选择** | 偶像团、校园、古风仙侠、职场等预设世界观 | - |
| **角色设定** | MBTI、性格标签、外貌偏好等个性化设定 | 乙女游戏 |
| **沉浸式漫画** | 条漫形式阅读，第一视角叙事 | Episode |
| **选择节点** | 关键剧情处暂停，呈现2-4个选择 | Choices |
| **管家互动** | 专属NPC管家，解锁陪伴对话 | 恋与制作人短信系统 |
| **结局收集** | 不同选择触发不同结局，可回溯重玩 | 视觉小说 |

## 五、技术方案

### 5.1 技术栈选型

| 层级 | 技术 | 理由 |
|-----|------|------|
| 前端 | Next.js 14 + React | SSR、路由、生态 |
| 样式 | TailwindCSS | 快速开发 |
| 状态 | Zustand | 轻量、简单 |
| 数据 | JSON/本地 → 后续 Supabase | MVP 先用静态数据 |
| 部署 | Vercel | 免费、快速 |

### 5.2 数据结构设计

```typescript
// 世界观
interface Universe {
  id: string;
  name: string;          // "偶像企划"
  cover: string;         // 封面图
  description: string;
  stories: Story[];
}

// 故事/剧本
interface Story {
  id: string;
  title: string;
  scenes: Scene[];
}

// 场景
interface Scene {
  id: string;
  type: 'comic' | 'choice' | 'ending';
  panels?: ComicPanel[];     // 漫画面板
  choices?: Choice[];        // 选择项
  ending?: Ending;           // 结局信息
}

// 选择
interface Choice {
  id: string;
  text: string;
  nextSceneId: string;       // 跳转到哪个场景
  effect?: {
    relationship?: number;   // 好感度变化
    flag?: string;           // 剧情flag
  };
}

// 漫画面板
interface ComicPanel {
  image: string;
  dialogue?: {
    character: string;
    text: string;
  };
}
```

## 八、竞品分析摘要

| 产品类型 | 陪伴感实现 | 沉浸感实现 | 我们的机会 |
|---------|-----------|-----------|-----------|
| Episode/Choices | 角色对话、玩家作为主角 | 分支选择、第一人称 | 漫画形式更视觉化 |
| 乙女游戏 | 全天陪伴、模拟社交 | 高品质美术声优 | 更轻量、碎片化 |
| Bubble | 伪私信、实时互动 | 艺人真实参与 | 不依赖真人艺人 |
| Character.AI | 持续对话、记忆系统 | 开放式交互 | 有剧情结构、更有仪式感 |

**差异化定位**: 漫画形式 + 追星设定 + 分支剧情 + UGC创作
